# 新版课程导航结构草案

> 状态说明：
> - 本稿讨论“玩家层导航骨架”。
> - 当前有效玩家层结构，以 `feasibility_layering.md` 为准。
> - 本稿中凡与 `feasibility_layering.md` 冲突的地方，以后者为准。

本草案用于下一阶段的课程重排与导航重构讨论。

目标不是“把更多章节塞进现有菜单”，而是把课程组织方式升级成一个适合中长线扩展的学习地图。

补充说明：
- 本稿主要讨论“导航骨架”。
- 最新的“孩子视角主线压缩版”见 `mainline_narrative_reframe_draft.md`。
- 后续导航实现应同时参考：
  - 本稿的顶层入口与信息架构
  - `chapter_blueprints_master.md` 的设计层单元
  - `mainline_narrative_reframe_draft.md` 的玩家层叙事

## 1. 为什么要重构导航

当前左侧汉堡菜单的问题不只是“章节变多”，而是信息架构快失效了：

- 课程继续增加后，平铺章节列表会越来越长。
- 主线课、挑战课、自由实验、方程工坊、家长入口会混在一起。
- 孩子需要的是“下一步去哪”，不是看到一长串章节编号。
- 家长需要的是“孩子正在学哪一类能力”，不是只看到很多关卡名。

因此下一步不应继续沿用“全局平铺章节清单”。

## 2. 参考教学思路后得到的结构原则

### 2.1 等号应先作为“关系”来组织课程
成熟资料普遍强调：等号不能先教成“算出答案的动作”，而应先教成“左右关系相等”。

这意味着导航也不应该围绕“技巧列表”组织，而应围绕“关系理解 -> 变形 -> 建模”组织。

### 2.2 小学代数应呈连续进阶，而不是突然跳到形式化方程
成熟资料普遍强调“algebraic thinking”的连续性：
- 关系
- 结构
- 等价
- 未知量
- 用故事表达关系

这支持把主线拆成几个清晰阶段，而不是所有章节同级并列。

### 2.3 应用题与图示模型是通向方程的桥梁
条形图 / 线段图 / 比较模型被大量成熟教学体系用作：
- 读懂数量关系
- 找“谁比谁多”
- 找总量、差量、倍数关系
- 过渡到方程表达

所以导航中要为“建模与辅助视图”预留位置。

### 2.4 课程地图应该服务两类用户
- 孩子：我接下来玩什么
- 家长：孩子正在建立什么能力

因此导航不应只给一种“工程视角”。

### 2.5 solve 关也要承担“读题习惯”培养
真实使用观察表明：
- 很多孩子在 solve 关里几乎不看题干
- 会直接开始拖拽和试错
- 到 build 关第一次真正需要读题时，阻碍会明显放大

因此未来课程蓝图应加入一个明确约束：
- 不能长期允许 solve 关只靠“看等式 + 狂移项”通关
- 要逐步在 solve 关中加入轻读题动作或半建模动作
- build 关不应成为孩子第一次被迫认真读题的地方

## 3. 新的顶层导航建议

建议把当前系统重组为 5 个顶层入口。

### 3.1 主线闯关
这是核心入口。

用途：
- 给孩子正常学习使用
- 给家长看到孩子当前阶段

内容：
- 按阶段组织的主线课程
- 每阶段若干单元
- 每单元若干关卡

### 3.2 挑战线
用途：
- 放奥数味 / 提高题
- 不打断主线节奏

内容：
- 逆向思考
- 和差倍
- 年龄
- 盈亏与平均
- 以后可加规律、假设法

### 3.3 方程工坊
用途：
- 从故事中选择未知数
- 拼关系
- 生成等式
- 再进入求解

这是“从会解到会造方程”的桥梁。

### 3.4 自由实验
用途：
- 输入方程
- 试各种变形
- 自己做模拟练习

### 3.5 家长入口
用途：
- 看课程说明
- 提交反馈
- 提交题目想法
- 了解当前学到哪

## 4. 主线闯关的内部结构建议

主线不再直接显示所有章节，而改成“阶段 -> 单元 -> 关卡”三级结构。

### 阶段 A：平衡感
目标：
- 理解等号表示一样多
- 理解未知量
- 认识加减变化

设计层单元：
- `A0 真正的等号初见`
- `A1 认识未知量`
- `A2 加减变化与常数请走`

玩家层建议名称：
- 第一程：看懂一样多

### 阶段 B：变形感
目标：
- 形成合法变形直觉
- 理解几个一样的未知量
- 理解平均分和分数系数

设计层单元：
- `A-Bridge 列式初见`
- `B1 几个一样的未知数`
- `B3 半份与分数系数`
- `B-Bridge 乘法 / 分数列式`
- `D1 括号是一整包`（已从第二程移出，作为独立过渡段）

玩家层建议名称：
- 第二程：让未知数更清楚

### 阶段 C：建模感
目标：
- 从故事中找未知量
- 把关系写成等式
- 用等式回到故事

设计层单元：
- `C1 故事到等式`
- `C2 选符号与干扰条件`
- `C3 选择未知量`
- `C4 方程工坊入门`（课程层保留，玩家层第一版冻结）

玩家层建议名称：
- 第三程：把故事变成方程

### 阶段 D：综合感
目标：
- 处理两边都有未知量
- 进入挑战线前的桥梁
- 把前面学过的内容整合起来

设计层单元：
- `D2 两边都有未知数`
- `D4 线段图桥接`（课程层保留，玩家层第一版冻结）
- `D3 主线方法地图复习`（课程层保留，玩家层第一版并入 D4 尾声）

玩家层建议名称：
- 第四程：看懂复杂关系

### 4.5 当前阶段结构与玩家叙事的关系
这里建议明确采用“双层结构”：

- 对内：
  - 保留 `A0 / A-Bridge / C3 / C4 / D3 / D4` 这类详细设计层单元
- 对外：
  - 不直接暴露所有内部桥接章节
  - 优先显示 4 段主线旅程

这不是为了“包装好看”，而是因为：
- 孩子更容易感知连续旅程
- 家长更容易理解当前在学哪类能力
- 设计层仍然可以继续诚实保留真正的桥接节点

### 4.6 当前已收口的玩家层版本
在当前实现与近期范围内，玩家层应读成：
- 第一程：看懂一样多
- 第二程：让未知数更清楚
- `D1`：第三程与第四程之间的独立过渡段
- 第三程：把故事变成方程
- 第四程：看懂复杂关系（第一版先只承载 `D2`）

也就是说，`C4 / D4 / D3` 目前不应作为第一版玩家层正式开放单元。

## 5. 挑战线的组织方式建议

挑战线不建议再按纯编号章节堆叠，而应按“思维模型”分组。

### 模块 1：逆向思考
- 神秘机器
- 倒推
- 运算链条

### 模块 2：关系题
- 和差
- 倍数
- 比较

### 模块 3：时间与变化
- 年龄
- 若干年后 / 若干年前

### 模块 4：分配与平均
- 盈亏
- 平均数
- 分组问题

未来如果继续增加：
- 规律与表达式
- 枚举与假设法

## 6. 线段图 / 条形图在导航里的位置

当前不建议把线段图做成与主线并列的独立大入口。

更合理的位置是：

### 6.1 在主线 D 阶段首次引入
孩子已经有等式基础后，再看到：
- 为什么“比较关系”适合画成线段
- 为什么“总量 + 差量”能转成图

### 6.2 在方程工坊里作为辅助视图
故事题可选：
- 直接列方程
- 先看线段图

### 6.3 在挑战线里作为解释层
对和差倍、年龄、盈亏类题：
- 先看图示关系
- 再转为方程

所以：
- 线段图是解释器
- 不是当前阶段的独立主玩法
- 第一版玩家层仍然冻结 `D4`，这里只保留未来导航位置

## 7. 菜单交互建议

### 7.1 汉堡菜单不再直接展开所有关卡
建议左侧菜单只放顶层入口：
- 主线闯关
- 挑战线
- 方程工坊
- 自由实验
- 家长入口

### 7.2 主页面用“学习地图卡片”承载内容
进入某个入口后，再看到阶段卡片或模块卡片。

例如进入主线：
- 阶段 A 平衡感
- 阶段 B 变形感
- 阶段 C 建模感
- 阶段 D 综合感

点开后再进入单元列表。

### 7.3 给孩子和家长不同的信息密度
孩子视图：
- 大卡片
- 阶段名
- 奖章 / 进度

家长视图：
- 阶段能力说明
- 当前卡点
- 建议复习点

不一定要做双模式切换，但信息层应允许这样演进。

## 8. 与当前课程的映射建议

当前已有章节可先粗映射为：

### 主线闯关
- 现第 1-4 章 -> 第一程 / 第二程 的早段
- 现第 5-6 章 -> 第三程
- 现第 7-8 章 -> 第四程与挑战线桥接

### 挑战线
- 现第 9-11 章 -> 挑战线基础模块
- 以后新增的挑战章继续归入挑战线，而不是挤进主线

这样做的好处：
- 现有关卡资产不必立刻大重写
- 只是先换组织方式
- 后续可逐步把旧章节重构成“阶段 + 单元”

## 9. 推荐的下一步落地顺序

### 第一步
先做“导航结构草图”而不是实现。

确认：
- 顶层入口名称
- 主线四阶段是否成立
- 挑战线四模块是否成立

### 第二步
做课程映射表。

把现有关卡逐个挂到：
- 顶层入口
- 阶段 / 模块
- 单元

### 第三步
先做“设计层 -> 玩家层”的主线压缩。

确认：
- 哪些单元在孩子视角中不单独显示
- 4 段主线旅程是否足够稳定
- 第二程是否过长

### 第四步
再讨论界面实现。

这时才决定：
- 汉堡菜单里怎么显示
- 首页是否改成地图卡
- 是否需要阶段封面页

## 10. 当前版本的明确建议

当前我不建议：
- 继续往现有左侧章节清单里无上限加章节
- 现在就把数轴和线段图都做成独立主模块
- 还没重组课程信息架构就继续大量加题

当前我建议：
- 先重构课程信息架构
- 先把主线与挑战线分开
- 先把线段图定位成辅助视图
- 先把设计层主线压缩成孩子能感知的 4 段叙事
